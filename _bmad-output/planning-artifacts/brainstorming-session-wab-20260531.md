## 1. 产品愿景（一句话宣言）

> **「错题银行」是一个父亲为自己孩子手作的、可生长 12 年的、AI 驱动的家庭学习陪伴系统——
> 它把每一道错题变成一只可被打败、收藏、进化的怪物，
> 把每一次复习变成爸爸/妈妈/孩子之间的 5 分钟仪式，
> 让孩子从「答题者」变成「侦探」、「猎人」、「出题者」。**

| ID | 约束 | 设计含义 |

- **长期资产** — 12 年生命周期（小学 → 高中），数据模型必须为长期画像设计
- **多端访问** — Web + Android（PWA 优先策略可大幅降本）

## 3. 第一性原理（六公理 + 推论）

### 3.1 六公理

| # | 公理 |
|---|---|
| 公理 1 | 错题的价值不在题本身，**在它指向的「知识盲点 / 思维陷阱 / 习惯漏洞」** |
| 公理 2 | **遗忘是生理常数**，不是态度问题 → 必须用 SRS 算法在临遗忘时唤醒 |
| 公理 3 | 真掌握 = **换情境 / 数字 / 问法依然能做对** → 必须有变式题 |
| 公理 4 | **错因 > 答对**（元认知） → 错因标签是最珍贵字段 |
| 公理 5 | **内驱力 > 外部督促** → 必须有趣、有掌控感、有成就反馈 |
| 公理 6 | 12 年留存的不是题，是**「思维成长史」** → 数据模型为长期画像设计 |

### 3.2 推论出的核心数据骨架

```
错题 (Mistake)
  ├─ 原题 (题干 + 选项 + 答案 + 孩子作答 + 错题快照图)
  ├─ 知识点节点 (Knowledge Point)    ← 跨学期可链接
  ├─ 错因标签 (Error Tag)             ← 强制必填，AI 兜底首填
  ├─ 自定义 Tag (Custom Tag)          ← 逻辑题 / 几何题等家长自定义
  ├─ 解题思路 (Solution Hint)          ← 家长配置 + AI 兜底
  ├─ 变式题集合 (AI Variants)          ← 5–10 道
  ├─ 复习记录 (Review History)         ← 喂给 SRS 算法
  └─ 掌握度 (Mastery Score)            ← 可视化"成长画像"
```

**关键差异**：复习单位 = **「知识点 + 错因簇」**，而非单题。

## 5. 录入侧：拍照识别技术路线

### 5.1 核心难点

| 难点 | 严重程度 |
|---|---|
| 手写体识别 | ⭐⭐⭐ |
| 数学公式（分数/根号/上下标） | ⭐⭐⭐⭐ |
| 几何图形 | ⭐⭐⭐⭐⭐ |
| 混排版式（题干+答案+批注红勾红叉） | ⭐⭐⭐ |
| 结构化抽取（区分题干/选项/答案/作答/错因） | ⭐⭐⭐⭐ |

### 5.2 选型：路线 C 主力 + 路线 B 兜底

| 路线 | 方案 | 角色 |
|---|---|---|
| **C - 多模态大模型** | GPT-4o / Claude 3.5 Sonnet / Qwen-VL-Max / GLM-4V | **主力**：一步输出结构化 JSON |
| **B - 商业题目识别 API** | 腾讯云试题识别 / 百度试卷分析 / MathPix（公式） | 兜底：复杂公式/特殊场景 |
| A - 开源 OCR | PaddleOCR / Tesseract | 不采用（覆盖不全） |

### 5.3 核心 Prompt 设计（伪代码）

```
你是一位资深小学老师。看这张错题照片，输出 JSON：
{
  "subject": "数学",
  "grade_guess": "一年级",
  "question_type": "应用题",
  "stem": "题干文本",
  "options": [...],
  "correct_answer": "...",
  "child_answer": "孩子作答（如能识别）",
  "image_blocks": ["geometry_figure_1.png 用于图形单独存储"],
  "auto_tags": ["20以内加法", "进位"],
  "error_hypothesis": "推测错因：进位忘加",
  "solution_hint_draft": "解题思路初稿（家长可二次编辑）"
}
```

### 5.4 三个录入入口

1. **单题拍照** — 即时拍 + 即时识别 + 即时确认
2. **整张试卷扫描** — A3/A4 试卷一张图，AI 自动切分为多题
3. **相册批量 / 微信文件导入** — 周末批量补录

## 6. AI 生题专项

### 6.1 六个生题维度

| 维度 | 说明 |
|---|---|
| D1 难度梯度 | 简单 / 等难度 / 进阶 / 综合 四档 |
| D2 题型变化 | 选择↔填空↔解答↔判断；Cloze 关键步骤挖空 |
| **D3 错因针对性** ⭐ | 抓具体错因（如"进位忘加"）专门设陷阱反复练 |
| D4 情境迁移 | 抽象题套孩子兴趣（我的世界 / 鹦鹉 / 车 / 汪汪队 / 冰雪奇缘） |
| D5 逆向生成 | 给答案让孩子出题；给条件让孩子设问 |
| D6 关联拓展 | 上下年级版本 / 跨学科融合 |

**强制规则**：每道生成题必带「为什么生成这道」的解释，防黑箱、便于家长审核筛选。

### 6.2 三个玩法

| 玩法 | 描述 |
|---|---|
| 🎯 错因猎人 | 系统盯紧某错因，每天偷塞 1 题进复习包，连续 N 天 0 出现 → 解锁徽章 |
| 🤺 AI 教练对决 | 答对升档生成下一题，看能连续撑几关，出 PR 个人最佳 |
| 🔄 角色互换出题 | 孩子向 AI 出"同类型考倒它"的题，AI 答 + 评价孩子出题水平 |

### 6.3 PDF 三模版

| 模版 | 描述 |
|---|---|
| 📄 每日小测 | 5–10 题 / A4 一页 |
| 📄 专项突破卷 | 围绕一个错因/知识点，15–20 题 |
| 📄 学期成长档案 | 错题 + 变式 + 掌握度报告，留作纪念 |

## 7. 复习侧：八玩法 + 优先级排序

### 7.1 八个玩法详情

| # | 玩法 | 机制简述 | 借招来源 |
|---|---|---|---|
| 1 | 🃏 经典翻牌（Anki 派） | 题正面 → 翻面 → 自评「记得/卡了/忘了」喂 SRS | Anki / Quizlet |
| 2 | ⏱ 速度赛 PR 挑战 | 60 秒内答对 X 题，破 PR 撒花 | Strava + Pokémon Stadium |
| 3 | 🐉 Boss 战五连关 | 同知识点 5 题难度递增，损 3 心重来 | 多邻国 + 塞尔达神庙 |
| 4 | 🔍 错因侦探（Bug 找茬） | 看错题过程，**只找错在哪步**，不重做 | Code Review + 逆转裁判 |
| 5 | 🤝 家庭对战 / 协作 ⭐ | 爸妈 vs 孩子各自答 / 三步协作 / 互出考题 | Wordle 家庭群 + 双人成行 |
| 6 | 🗺 主题地图探险 | 错题挂在《我的世界》/《汪汪队》/《冰雪》风格地图节点 | Pokémon GO + 我的世界 |
| 7 | 🎙 语音口述题 | AI 念题 + 孩子语音答 | 得到 + Khanmigo |
| 8 | 🎭 兴趣化剧情包装题 | 题目实时套孩子兴趣场景（D4 落地形态） | Khan 名人例子 + 早教 IP |

### 7.2 优先级排序

```
🥇 P0 MVP 三件套：
   - 玩法 1（经典翻牌）
   - 玩法 5（家庭对战）⭐ 高黏性核心
   - 玩法 8（兴趣化剧情）

🥈 P1 V1.0 加固：
   - 玩法 3（Boss 战）
   - 玩法 6（主题地图探险）
   - 玩法 4（错因侦探）

🥉 P2 后续：
   - 玩法 2（速度赛 PR）
   - 玩法 7（语音口述题）

⚠️ 注：TTS 题干朗读（玩法 7 的子能力）作为底层能力（D-01）嵌入所有 P0 玩法
```

### 7.3 节奏设计（基于工作日/周末区分）

```
周一 ~ 周五：
  - 录题（每天作业批改后）
  - 重点复习 5–10 题（玩法 1 + 玩法 8）
  - Wordle 式每日一题（玩法 5 简化版）

周末：
  - 错题深度复习
  - Boss 战 / 挑战模式（玩法 3 + 玩法 6）
  - 家庭对战大场（玩法 5 完整版）
```

## 11. TTS 技术方案（D-01 详细版）

### 11.1 路线选型

| 路线 | 方案 | 月成本（自家用 ~1500 次） | 角色 |
|---|---|---|---|
| **B - Edge-TTS（开源白嫖）** ⭐ | `edge-tts` Python 库调用微软 Azure 神经语音 | **¥0** | **主力** |
| A - 浏览器 Web Speech API | 浏览器内置 SpeechSynthesis | **¥0** | 兜底 |
| C - 云厂商 API | 腾讯/阿里/Azure | **¥0**（免费额度内） | 备选 |

### 11.2 默认配置

| 项目 | 值 |
|---|---|
| 引擎 | Edge-TTS（服务端） |
| 默认音色 | `zh-CN-YunxiaNeural`（云夏温柔童声） |
| 默认语速 | `rate="-10%"` |
| 缓存策略 | 同文本+同音色 → MP3 落本地磁盘 / OSS，永久缓存 |
| 失败降级链 | Edge-TTS → 云厂商 API → 浏览器 Web Speech |

### 11.3 文本预处理（朗读规范化）

| 原文 | 朗读化 |
|---|---|
| `3+5=?` | "3 加 5 等于多少" |
| `2×4` | "2 乘以 4" |
| `5/8` | "8 分之 5" |
| `3²` | "3 的平方" |
| `√16` | "根号 16" |

### 11.4 开发量估算

- 后端（Edge-TTS 微服务 + 缓存层）：**1 天**
- 前端（朗读按钮 + 音色选择 + 播放控件）：**0.5 天**
- 文本预处理（数学符号正则规则 + LLM 兜底）：**0.5–1 天**
- **合计：2–3 人日**
